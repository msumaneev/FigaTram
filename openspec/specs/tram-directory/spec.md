## ADDED Requirements

### Requirement: Ведение справочника трамваев
Система MUST предоставлять локальную базу данных (Room) для хранения справочной информации о трамваях (бортовой номер, описание, модель, URI фотографии, флаг "любимый").

#### Scenario: Сохранение данных о трамвае
- **WHEN** пользователь вносит информацию о трамвае и прикрепляет его фото
- **THEN** система сохраняет эту информацию в БД для дальнейшего использования

### Requirement: Просмотр списка трамваев
Система MUST иметь отдельный экран "Справочник", где отображается список всех известных системе трамваев с их миниатюрами.

#### Scenario: Открытие справочника
- **WHEN** пользователь переходит во вкладку "Справочник"
- **THEN** отображается список всех ранее сфотографированных и описанных трамваев

### Requirement: Интеграция карточки с картой
Данные из справочника MUST использоваться при открытии карточки трамвая на карте (отображать сохраненное фото и описание, если трамвай уже есть в справочнике).

#### Scenario: Просмотр известного трамвая на карте
- **WHEN** пользователь кликает на трамвай, который есть в справочнике
- **THEN** в карточке подгружается сохраненная фотография и информация

### Requirement: Избранные трамваи (Любимые)
Система MUST позволять пользователю отмечать галочкой (добавлять в избранное) определенные трамваи в справочнике. Отслеживание ETA и уведомления будут применяться к этим "любимым" трамваям (а также к инклюзивным).

#### Scenario: Выбор любимого трамвая
- **WHEN** пользователь ставит галочку "любимый" на карточке трамвая в справочнике
- **THEN** система сохраняет этот статус в БД и включает этот трамвай в систему отслеживания для Push-уведомлений
