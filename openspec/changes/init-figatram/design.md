## Context

Инициализация нового Android-проекта FigaTram для отслеживания трамваев в Анталье (линия T3). Приложение требует интеграции Google Maps, поллинга неофициального API с частотой 20 секунд, реализации фоновой задачи для расчета ETA (для инклюзивных трамваев 152 и 175) и отправки локальных Push-уведомлений. Также добавляется функционал "справочника споттера": возможность просматривать все трамваи на карте и открывать карточку с их характеристиками/фотографиями.

## Goals / Non-Goals

**Goals:**
- Создание масштабируемой архитектуры MVVM (Model-View-ViewModel) + Clean Architecture.
- Плавное отображение **всех** трамваев на Google Maps (обновление каждые 20 сек).
- Интерактивность карты (клик по маркеру открывает карточку трамвая в стиле Flightradar).
- Создание и ведение справочника трамваев (фотографии, описания) в локальной БД.
- Надежная фоновая работа (отправка локальных пуш-уведомлений в нужное время).
- Сбор и сохранение локальной статистики прибытия целевых трамваев через Room.

**Non-Goals:**
- Реализация собственного бэкенда (используется существующий внешний API).
- Мультиплатформенность (iOS/Web не входят в скоуп).
- Публикация в Google Play на текущем этапе.

## Decisions

- **UI Framework**: Jetpack Compose (Material Design 3). Для карточки трамвая будет использован `BottomSheet` (ModalBottomSheet).
- **Сетевой слой**: Retrofit2 + OkHttp3 с отключенным SSL Pinning.
- **Карты**: Google Maps SDK for Android (`maps-compose`).
- **Фоновая работа**: Использование `Foreground Service`.
- **Локальная БД**: `Room Database` используется для хранения аналитики и **справочника трамваев** (таблицы `ArrivalStats`, `TramInfo`). В `TramInfo` будут храниться локальные пути к сфотографированным изображениям и текстовые данные.

## Risks / Trade-offs

- **Потребление батареи**: Отображение всех маркеров и поллинг. *Смягчение*: оптимизация отрисовки маркеров, отключение поллинга при сворачивании приложения (кроме Foreground Service для ETA).
- **Хранение фото**: Локальные фото могут занимать много места. *Смягчение*: сохранение сжатых копий фото (напр. WebP) или использование URI галереи.
